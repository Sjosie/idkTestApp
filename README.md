# idkTestApp

# Архитектура
Приложение построено вокруг простого BLoC, отвечающего за состояние подписки и её сохранение в SharedPreferences.
Во всём дереве виджетов BLoC пробрасывается через BlocProvider в MyApp, что позволяет обращаться к состоянию подписки с любого экрана.
Начальный экран выбирается на основе сохранённого флага подписки: если подписка есть — сразу открывается MainScreen, иначе показывается OnboardingFlow.

Логика подписки инкапсулирована в SubscriptionBloc, который реагирует на событие BuySubscription и сохраняет состояние (has_subscription и subscription_type) в SharedPreferences.
UI-слой максимально «тонкий»: экраны дергают только события BLoC и выполняют навигацию, не зная деталей сохранения.

# Структура проекта
В тестовом варианте вся логика находится в одном файле lib/main.dart для упрощения ревью кода.

Основные части:

main() и MyApp — точка входа, инициализация SharedPreferences, создание SubscriptionBloc, настройка MaterialApp и выбор стартового экрана.

BLoC: SubscriptionType, SubscriptionEvent/BuySubscription, SubscriptionState, SubscriptionBloc с обработчиком _onBuySubscription.

Онбординг:

OnboardingFlow — PageView с двумя экранами, кнопка «Продолжить» либо листает страницы, либо ведёт на PaywallScreen.

_OnboardingPage — простой экран с заголовком и описанием.

Paywall:

PaywallScreen — два варианта подписки (месяц/год) с выбором через StatefulBuilder, кнопка «Продолжить» отправляет событие BuySubscription и делает pushAndRemoveUntil на MainScreen.

_PlanTile — карточка тарифа с выделением выбранного плана.

Главный экран:

MainScreen — ListView.separated с простым текстовым контентом, демонстрирующим работу приложения после покупки подписки.

# Что бы улучшил при большем времени
Разделение по папкам и файлам
Вынес бы слои в отдельные директории: lib/bloc/subscription_bloc.dart, lib/ui/onboarding, lib/ui/paywall, lib/ui/main, чтобы код не концентрировался в одном файле.

Чистая архитектура / слои
Добавил бы слой use case/репозитория для работы с подпиской, вместо прямого обращения BLoC к SharedPreferences, чтобы хранение можно было легко заменить (например, на secure storage или локальную БД).

Стейт-менеджмент и тесты

Расширил бы состояние BLoC (статусы загрузки, ошибки, тип подписки) и добавил бы unit‑тесты на логику сохранения и восстановления подписки.

Для paywall сделал бы отдельный BLoC/Cubit, чтобы не смешивать выбор тарифа и сам факт покупки с глобальным стейтом подписки.

Навигация и роутинг
Внедрил бы декларативный роутинг (например, go_router) вместо явных Navigator.push/pushAndRemoveUntil, чтобы легче описывать флоу «онбординг → paywall → главный экран» и его ветвления.

UI/UX улучшения

Добавил бы более продуманный дизайн paywall: иконки, визуализацию скидки, анимации выбора тарифа.

Сделал бы локализацию (EN/RU) и вынес бы строки в отдельный intl/l10n слой.
